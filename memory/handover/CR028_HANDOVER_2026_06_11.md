## What the owner wants
1. Place Order + Collect Bill payloads must send **item-level discount** — `discount_value` (raw input) + `discount_amount` (actual ₹ after discount). Today every item sends hardcoded `discount_amount: '0.00'` and no `discount_value`.
2. Items flagged **`give_discount = 'No'`** must be **excluded** from discount entirely.
3. Coupon item/category scope: when CRM targets specific items, discount goes ONLY to those items — not spread uniformly.

## Key design decisions LOCKED by owner
- Coupon and manual/preset are **mutually exclusive** (code-verified — UI blocks one when the other is active).
- `discount_type` **NOT sent at item level** — backend derives from order-level `order_discount_type`.
- `discount_amount` at item level = actual ₹ amount AFTER discount (NOT the discount itself).
- `discount_value` at item level = raw input (10 for 10%, prorated ₹ for flat/coupon).
- Same keys (`discount_value`, `discount_type`) exist at both levels with DIFFERENT meanings — see CR doc §3.
- BOGO: `food_amount` = real menu value, `discount_value` = food_amount (100% off), `discount_amount` = `"0.00"`.
- CRM coupon does NOT override `give_discount='No'` — coupon rejected if target item is non-discountable.
- Flat ₹ discount capped at `discountableTotal`.
- POS Frontend distributes (largest-remainder) — not backend.
- Trust CRM `benefit_items` values; edge guards E-1 through E-5 in CR doc.

## Investigation results you can trust (code-verified 2026-06-11)
- Discount is **order-level only** today: one formula on aggregate `itemTotal` of `billableItems`.
  - Place Order / QSR: `CartPanel.jsx:384-387` (presetDiscount, manualDiscount on itemTotal).
  - Collect Bill: `CollectPaymentPanel.jsx:514-527` (+ coupon/loyalty/wallet), `discountRatio` GST proration at `:579-583`.
- Per-item payload: `orderTransform.js:603` → `discount_amount: '0.00'` hardcoded in `buildFoodItem()`. `orderTransform.js:1345` → same in `food_detail[]` for collectBillExisting.
- `give_discount` flag lives in POS menu API (key: `give_discount`, values: Yes/No) AND menu management (`menuManagementTransform.js:95/245`). **`productTransform.js` does NOT map it** → cart items never have it.
- Coupon/manual mutual exclusivity: `CollectPaymentPanel.jsx:1318-1319` (`couponBlocked = isManualActive`) + `BUG108_COPY.couponBlockedByDiscount` / `discountBlockedByCoupon`.
- Exclusions that DO work today (copy this pattern — BUG-018 complimentary carve-out): cancelled, complimentary (catalog+runtime), check-in markers, room-balance pass-through.
