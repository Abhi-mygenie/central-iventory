## 1. Unified "Download / Share" control (NEW — replaces separate Export buttons)

### 1.1 Single trigger button

- **Label:** `Download` (with chevron-down)
- **Icon:** `Download` from `lucide-react`
- **Placement:** Top-right of the report header strip, in line with the date filter + Apply
  button (per the loading-spec disable rules — disabled while fetch is in-flight)
- **Style parity:** identical button-shell to Credit module's `Portfolio Summary` button —
  - `inline-flex items-center gap-1.5 px-3 py-2 rounded-lg text-xs font-semibold border`
  - Border + text in `COLORS.primaryOrange`, hover `bg-orange-50`
  - `disabled:opacity-40 disabled:cursor-not-allowed` when fetch in-flight or `filtered.length === 0`
- **`data-testid`:** `report-download-trigger-btn`

### 1.2 Menu options (opens a dropdown / popover)

| # | Option | Icon (lucide) | State at ship | Test ID |
|---|---|---|---|---|
| 1 | **Download as Excel** | `FileSpreadsheet` | ENABLED — produces XLSX matching all currently-visible tabs (Combined Export) | `report-download-excel-btn` |
| 2 | **Download as PDF**   | `FileDown`        | ENABLED — produces print-style PDF matching all currently-visible tabs                  | `report-download-pdf-btn` |
| 3 | **Send via Email (attachment)** | `Mail` | **DISABLED** (placeholder) — tooltip: "Coming soon — Email attachment (Phase 2B)" | `report-share-email-btn` |
| 4 | **Send via WhatsApp** | `MessageCircle`   | **DISABLED** (placeholder) — tooltip: "Coming soon — WhatsApp integration (Phase 2B)"   | `report-share-whatsapp-btn` |
| 5 | **Send via SMS**      | `Send`            | **DISABLED** (placeholder) — tooltip: "Coming soon — SMS integration (Phase 2B)" | `report-share-sms-btn` |

Disabled items follow the Credit module's `ActionIconBtn` pattern:
- `disabled` attribute set
- Tinted-grey icon + label
- `cursor-not-allowed`
- Tooltip on hover surfacing the phase string

### 1.3 Behaviour

- Clicking the trigger opens the dropdown anchored to the button (`Popover` from `@radix-ui/react-popover`, consistent with the rest of CR-011 primitives).
- The dropdown is keyboard-navigable (arrow keys, Enter to activate, Esc to close).
- The three disabled rows (Email, WhatsApp, SMS) are reachable by keyboard but do not fire `onClick`.
- Selecting Excel / PDF emits an event the host screen handles (the actual generation logic
  ships at Code Gate 2; the mockup wires a `console.log` stub).

### 1.4 Replaces

This unified control **replaces** the previously planned "Combined Export" button on S5.
"Combined Export" semantics are preserved (one file covering all 5 tabs), they just live
inside the Download menu's `Excel` and `PDF` options.

## 2. Format parity contract — "same across, like tab management"

The PDF and Excel outputs MUST use the same visual template style as the Credit / Tab
Management module's statement generator
(`/app/frontend/src/utils/creditStatementGenerator.js`). Specifically:

### 2.1 Header (every export)
- Restaurant logo top-left, restaurant name + address right of logo
- Report title (`Item Sales — Combined`) centre-top
- Date range pill (`from` → `to`)
- Generated-at timestamp right-aligned (IST, 24h)

### 2.2 Body
- Same font stack (system serif fallback) and colour palette (`COLORS.darkText`,
  `COLORS.primaryOrange`, `COLORS.sectionBg`, `COLORS.borderGray`)
- ₹ formatting via `formatINR` (Indian locale, 2 decimals; negative numbers prefixed with `-`)
- Date formatting `DD/MM/YYYY` (IST)
- Tab sections labelled with the same chip used on-screen (`All Items`, `Variation`, `Addon`,
  `Comp`, `Station`)
- Column headers in `COLORS.sectionBg` with uppercase `text-xs font-semibold`
- Row stripes optional (must be off by default to keep print-friendly)

### 2.3 Footer (every export)
- Page numbering bottom-right (PDF only)
- Restaurant name + report title bottom-left (for both formats)
- "Generated by MyGenie POS" centred (small, grey)

### 2.4 Excel-specific
- One sheet per tab (`All Items`, `Variation`, `Addon`, `Comp`, `Station`)
- Sheet 1 = `Summary` (totals + date range + restaurant info)
- Frozen header row + auto-width columns
- No formulas — flat values (matches credit statement principle of "snapshot, not workbook")

### 2.5 PDF-specific
- Generated via the same HTML→Blob→`window.open(URL.createObjectURL(blob))` pattern as
  `creditStatementGenerator.js` (no new dependency). Print-stylesheet `@media print` rules
  drive the actual PDF capture (user "Save as PDF" via browser print dialog).

### 2.6 Implementation note
- A shared utility `reportExporter.js` SHALL be created at Code Gate 2 under
  `/app/frontend/src/utils/` exposing `exportReportAsExcel(payload)` and
  `exportReportAsPDF(payload)`. It MUST reuse the formatter helpers (`fmtINR`, `fmtDate`,
  `BADGE_STYLES`, etc.) from `creditStatementGenerator.js` and follow the same
  HTML-template-string approach for PDF.
- This utility becomes a shared CR-011 primitive (alongside `<ReportLoadingShield>` and
  `useReportFetch`) and will be reused by **every** Phase-2/3/4 screen that exposes the
  Download menu. Per `CR_011_LOADING_AND_INTERACTION_SPEC.md §7`, primitives are built at
  the first Code Gate where they are needed (Code Gate 2 for `reportExporter.js`).

**UPDATE 2026-06-02 (owner directive "go path C"):** `reportExporter.js` was **promoted
to Code Gate 1.5** and SHIPPED today so that S5 Gate ⑤ can validate numbers AND export
format in a single owner pass. The utility lives at `/app/frontend/src/utils/reportExporter.js`
and exposes:

- `openReportWindow()` — synchronous popup opener (must be called in the click handler
  stack to avoid popup blockers)
- `exportReportAsPDF(win, payload)` — HTML→Blob→`win.print()` (same pattern as
  `creditStatementGenerator.js`; reuses `STATEMENT_CSS` + helpers)
- `exportReportAsExcel(payload, filename?)` — multi-sheet **SpreadsheetML 2003 XML**
  (`.xls`), dependency-free; opens natively in Excel / LibreOffice / Google Sheets.
  Includes: Summary sheet first (report metadata + KPIs), then one sheet per tab; frozen
  header row, styled header (orange), totals row at table bottom, ₹/integer/date/percent
  number formats.

Payload shape is payload-agnostic (any report can use it):
```js
{
  title, subtitle, restaurant: { name, address, id }, dateRange: { from, to }, generatedBy,
  kpis: [{ label, value, tone, format }],
  sheets: [{ name, subtitle, columns: [{ key, label, align, format, width }], rows, totals }],
}
```

Smoke-verified on S5 (Palm House, restaurant_id 541, April-2025 window):
- Excel: 16,209-byte file `Item_Sales_2025-04-01_2025-04-30.xls`, 6 worksheets
  (Summary + All Items + Top Sellers + Slow Movers + Cancelled Lines + Complimentary),
  12 styles, valid SpreadsheetML envelope.
- PDF: popup `Item Sales — The Palm House`, H1 `ITEM SALES`, 5 section headers each
  carrying its row count, KPI strip + period bar render per `STATEMENT_CSS` parity.
