## 1. Requirement

Customer needs a report for a selected period (month/year) where order data is **grouped station-wise**. Instead of showing the full combined invoice, the report **distributes/breaks down each order's items, subtotal, tax, and total across different kitchen stations**.

The report works like a **pseudo-invoice report per station** — showing how each order's revenue splits across stations.

---

## 2. Expected Behavior

- **New screen:** "Food Court" under Insights in sidebar
- **Route:** `/reports-module/food-court`
- **Layout:** Clones Order Ledger (S6) — S7 header, flat table, KPI strip, TOTAL row, search, column chooser, side-sheet drill, Excel/PDF export
- **One new top-level filter:** Station dropdown (one station at a time)
- When a station is selected:
  - Table shows only orders that have items from that station
  - Each order row shows only that station's items (name, qty, price charged)
  - Financial columns (Subtotal, GST, Total) reflect only the selected station's items
  - KPI strip + TOTAL row reflect the selected station's totals
- User reviews Station 1 report, then switches to Station 2, etc.
- **Tabs:** Simple — All / Settled
- **Data source:** `order-logs-report` → `order_details_table[].station_name`
