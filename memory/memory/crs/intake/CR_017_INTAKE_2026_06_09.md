## 1. Problem Statement (Owner Verbatim)

> There should be a way to send a WhatsApp message to the customer. For this flow, we will first call our API, which will generate and return a Razorpay payment link. After that, we will call another CRM API, which will send the WhatsApp template to the customer. All required variables, including the payment link and customer details, will be passed through the API.

**Clarification (brainstorm session):** Backend orchestrates both steps — POS makes a single API call. Backend creates (or reuses) Razorpay link, saves to `order_online_payments` table, and triggers WhatsApp via `razoar_payment_with_url` template. For existing pending links, backend reuses the stored link and sends SMS (WhatsApp resend currently commented out on backend side).

## 3. Backend API Contract (Confirmed)

### Endpoint
```
POST https://preprod.mygenie.online/api/v1/razor-pay/payment-link
Authorization: Bearer {auth_token}
Content-Type: application/json
X-localization: en
```

### Request Payload
| Field | Required | Notes |
|---|---|---|
| `order_id` | Yes | Must exist in orders table |
| `payment_amount` | Yes | Amount shown in WhatsApp message |
| `customer_phone` | Yes | 10-digit mobile number |
| `customer_name` | Optional | Defaults to "Customer" |
| `restaurant_name` | Optional | Falls back to restaurant name from DB |

### Response — Success (New Link)
```json
{
  "order_id": "850317",
  "payment_link": "https://rzp.io/i/xxxxx",
  "source": "razorpay"
}
```

### Response — Success (Existing Pending Link Reused)
```json
{
  "order_id": "850317",
  "payment_link": "https://rzp.io/i/xxxxx",
  "source": "db"
}
```

### Backend Behavior
1. **New link path:** Creates Razorpay payment link → saves to `order_online_payments` → sends WhatsApp via `razoar_payment_with_url` template
2. **Existing pending link path:** Reuses stored link → sends SMS via `sendPaymentLinkSms()` (WhatsApp resend block currently commented out on backend)
