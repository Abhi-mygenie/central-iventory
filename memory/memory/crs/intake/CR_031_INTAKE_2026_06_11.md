## 1. Requirement
One canonical money formula and date gate for cancellation loss, so Dashboard and Cancellations report agree.

## 2. Current State (verified divergence)
| Month (Palm House) | Cancellations report | Dashboard | Δ |
|---|---|---|---|
| Mar | 66,235 | 52,760 | 13,475 |
| Apr | 42,984 | 36,326 | 6,658 |
| May | 82,465 | 72,012 | 10,453 |

Verified causes (both audits):
1. **Money basis:** Cancellations = per-line `subtotal + tax`; Dashboard = `order_amount` (order-cancels) + raw `line.price` (item-cancels)
2. **Partially-cancelled orders:** `order_amount` ≠ item sums (cafe103 order 012580: ₹120 vs ₹10,680)
3. **Date gates:** Dashboard requires item `cancel_at` in calendar range on top of created_at fetch; Cancellations displays by `cancel_at` but filters by created_at business day
4. **Counting:** Dashboard counts lines (`+=1`); Cancellations counts qty

## 3. Expected Behavior
Both surfaces report the same loss number for the same period, from one shared cancellation-valuation helper.

## 4. Scope
- Files: `insightsService.js` (getDashboardAggregated cancel block), `CancellationsMockup.jsx`
- Define: canonical formula (proposal: per-line subtotal+tax, qty-counted, cancel_at-attributed) — owner sign-off on definition
