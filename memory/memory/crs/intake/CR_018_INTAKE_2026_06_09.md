## 1. Problem Statement (Owner Verbatim)

> The requirement is to schedule the order. So you can schedule it for next day or some other time. For schedule order send `schedule_at` like `2023-12-27 16:41:38` and `scheduled = 1`. Only these 2 keys add in payload.

**Backend code reference (Laravel):**
```php
$order->schedule_at = $datavalidation['schedule_at'] ?? null;
$order->scheduled = isset($datavalidation['scheduled']) ? (int)$datavalidation['scheduled'] : 0;
```

## 5. Scope

### In Scope
- Schedule toggle checkbox in CartPanel (next to KOT/Bill)
- Expandable date/time picker row (15-min intervals, up to 1 month)
- Pass `scheduled` and `scheduleAt` through `options` to `orderTransform.js` (both flows)
- "Scheduled" badge on OrderCard (showing date/time)
- Scheduled orders excluded from ScanOrderPopOut (not web origin, `scheduled=1` guard)
- New column in Status View for Scheduled + Reserved orders
- OrderEntry passes schedule state down to CartPanel and place-order call

### Out of Scope
- Delivery orders (no scheduling)
- Room orders (no scheduling)
- Edit scheduled time after placement (cancel and re-place)
- Auto-trigger/notification when scheduled time arrives (future CR)
- Backend changes to running orders API (backend team responsibility)
- QSR mode scheduling (follows same pattern but deferred)
