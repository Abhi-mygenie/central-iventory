# BUG-127 — Intake Document (Gate 0 + Gate 1)

**Status:** REGISTERED · **Priority:** P2 · **Sprint:** POS 4.0 · **Date:** 2026-06-11
**Source:** Insights audits — cafe103 (A6) + Palm House (A8)
**Type:** Bug

---

## 1. Symptom
Dashboard "Unsettled TAB" tile always shows ₹0, even with hundreds of TAB (credit) orders outstanding. The tile is misinformation.

## 2. Root Cause (verified)
`insightsService.js` getDashboardAggregated (~line 655):
```js
if (isPaid) {                       // requires fStatus === '6'
  ...
  if (pm.toLowerCase() === 'tab' && fStatus !== '6') {   // can never be true
    unsettledTab += orderAmount;
  }
}
```
The condition is nested inside the branch that requires `fStatus === '6'` — logically unreachable.

## 3. Verified Impact
₹0 in all months at both restaurants despite 319 TAB orders (Palm House) / 141 (cafe103).
**Compounding factor:** backend stamps ALL TAB orders `f_order_status=6` at punch (BUG-129), so even hoisting the condition out would still read ₹0 from this payload. The tile needs redefinition (e.g., `payment_method==='TAB'` outstanding value) or removal.

## 4. Scope
- Area: Reports / Insights — Dashboard
- File: `frontend/src/api/services/insightsService.js`
- Depends on: owner decision re TAB semantics (BUG-129 / CR-030 TAB policy)
