# BUG-126 — Intake Document (Gate 0 + Gate 1)

**Status:** REGISTERED · **Priority:** P2 · **Sprint:** POS 4.0 · **Date:** 2026-06-11
**Source:** Insights audits — cafe103 (A8.1) + Palm House (A7)
**Type:** Bug

---

## 1. Symptom
Items & Menu "Sold" revenue silently excludes round-off amounts; the report can never be footed exactly against Sales/Dashboard.

## 2. Root Cause (verified)
`insightsService.js` (getItemSalesAggregated, ~line 93):
```js
const orderRoundOff = parseFloat(ot.round_off) || 0;   // field does NOT exist
```
The API field is **`round_up`**. Confirmed on live payloads: `'round_off' in orders_table === false`, `'round_up' === true`. Round-off is therefore never distributed to item lines.

## 3. Verified Impact
| Restaurant | Missing per month |
|---|---|
| Palm House | Mar ₹460 · Apr ₹305 · May ₹272 · Jun ₹23 |
| cafe103 | ~₹100+/month |

## 4. Scope
- Area: Reports / Insights — Items & Menu
- File: `frontend/src/api/services/insightsService.js` (1 line)
- Fix: `ot.round_off` → `ot.round_up`

## 5. Evidence
- `INSIGHTS_REPORTS_AUDIT_PALMHOUSE.md` §A7 · `/app/audit_data/analyze.py` (`items.roundoff_missing`)
