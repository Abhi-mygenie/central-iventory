# BUG-125 — Intake Document (Gate 0 + Gate 1)

**Status:** REGISTERED · **Priority:** P1 · **Sprint:** POS 4.0 · **Date:** 2026-06-11
**Source:** Insights cross-report audits — cafe103 (A3, 2026-06-10) + Palm House (A4, 2026-06-11)
**Type:** Bug

---

## 1. Symptom
The Cancellations report (`/reports-module/cancellations`) "Order-Level" tab is **permanently empty**. Every order-level cancellation renders under "Item-Level" instead.

## 2. Root Cause (verified)
`CancellationsMockup.jsx` classifies scope with:
```js
const isOrderCancelled = pm === 'cancelled';   // pm = (payment_method || '').toLowerCase()
```
The live API value is **`'Cancel'`** → lowercased `'cancel'` ≠ `'cancelled'`. Verified on live data:
- cafe103: 41 cancelled orders Mar–Jun, all `payment_method='Cancel'`, **0 matched**
- Palm House: 259 cancelled orders Mar–Jun (109/49/97/4), **0 matched** (replication: order-scope qty 0/310, 0/185, 0/392, 0/23)

The Dashboard and the shared transform already classify correctly (`paymentMethod === 'Cancel' || fStatus === 3` in `reportTransform.deriveOrderStatus`).

## 3. Expected Behavior
Order-level cancels (whole order cancelled) appear under the "Order-Level" tab; item-level cancels under "Item-Level".

## 4. Scope
- Area: Reports / Insights — Cancellations
- File: `frontend/src/pages/reports-module/CancellationsMockup.jsx` (1 line)
- Fix direction: match `pm === 'cancel'` (or align with `fStatus === 3` / shared transform predicate)
